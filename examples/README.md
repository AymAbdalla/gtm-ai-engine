# Examples

What the system actually produces. **Everything here is synthetic**: generated by the production skills against a fictional account universe (Meridian Robotics and friends), so the structure and behavior are real and the data is 100% invented. No real companies, prospects, customers, or figures appear — see the note baked into each artifact. This is deliberate: redacting real work product leaks (metadata, missed cells) and looks like black bars; regenerating against fiction proves the system runs and publishes nothing.

Each artifact maps to a claim in the [main README](../README.md), with the thing worth noticing:

### [`scored-list-triage.xlsx`](scored-list-triage.xlsx) — the scorer's list mode
A 12-account triage: FIT cell color-coded green/keep, yellow/qualify, red/remove, with win heat, lookalike match, closest won customer, and a data-flag column. **What to notice:** two rows where the *file was wrong* (a stale headcount, a 10x mis-sized record) and one absorbed entity — caught because no account is graded off the uploaded file. The lean schema is intentional: colors and verdicts, no numeric score columns, because reps act on verdicts.

### [`triage-summary-meridian-list.md`](triage-summary-meridian-list.md) — the companion doc
Counts up top, accounts grouped by verdict, and 3–4 discovery questions under every borderline account. **What to notice:** borderline accounts don't get a shrug — they get the exact questions that would flip or DQ them, matched to *why* each one is borderline. And every remove carries its one-line reason; DQ is a first-class outcome, not an empty cell.

### [`account-plan-meridian-robotics.md`](account-plan-meridian-robotics.md) — the outreach engine's account plan
Snapshot, ranked triggers, value hypothesis, and a 9-seat buying-committee map. **What to notice:** every mapped name carries a confidence rating and a source; stale sources get a "verify first" flag; unfilled seats ship with the search strings to fill them; and the single-thread check is a launch gate, not a suggestion. Coverage is a tracked percentage.

### [`cold-email-pair.md`](cold-email-pair.md) — the persona router, caught in the act
Two Day-1 emails to the same fictional account: one to the CPO (HR protocol), one to the CFO (Finance protocol). **What to notice:** same triggers, same research — different subject, structure, vocabulary, proof treatment, and CTA mechanics, because exactly one protocol governs each message. Annotations under each email name the rules that fired.

### [`morning-brief.png`](morning-brief.png) — a synthetic render of the unattended morning brief
([`morning-brief.html`](morning-brief.html) is the source.) **What to notice:** every "needs attention" item is verified still-open before it lands; the reply card surfaces sequence state and *pauses the cadence* per rule; the departure card treats a lost contact as a new-in-seat trigger; and the footer states the boundaries — connectors and search only, nothing sent autonomously.
